Linux命令记录

  • man

    • 这条命令必须不顾字母顺序排在最前面。Linux提供了丰富的帮助手册，当你需要查看某个命令的参数时不必到处上网查找，只要man一下即可。 例如，man cat，当然，要进一步了解man的话，直接man man就可以啦。

  • cat

    • 简略的来说，cat主要有三大功能：

    关于cat
    功能 一次显示整个文件 从键盘创建一个文件 将几个文件合并为一个文件
    命令 cat filename cat > filename cat file1 file2 > file

    其中，从键盘创建文件时，只能创建新文件,不能编辑已有文件。当然，如果要详细了解cat的使用规则，是可以man cat的哈。

  • chmod
  • export

    • 设置或显示环境变量。以我最近的经历为例，我需要使用centos下安装的一个软件what（当然使用某个命令也是一样的道理）， 但这个软件/命令的执行文件不在当前目录，于是我每天创建并登陆虚拟机后都必须指定执行文件的目录，很麻烦， 如果在.bashrc中加入export LD_LIBRARY_PATH="xxx/yyy/..."，就相当于告诉程序，我需要使用what软件时， 需要的文件或相关的一些东西在这个目录里下。

  • echo

    • echo命令的功能是在显示器上显示一段文字，一般起到一个提示的作用。比如，上一条export命令的示例中， 可以使用$echo $LD_LIBRARY_PATH来查看LD_LIBRARY_PATH的内容。

  • grep
  • journalctl
  • ls

    • ls命令就是list的缩写，缺省参数时ls用来打印出当前目录的清单，如果ls指定其他目录，那么就会显示指定目录里的文件及文件夹清单。 通过ls命令不仅可以查看linux文件夹包含的文件，还可以查看文件权限(包括目录、文件夹、文件权限)，查看目录信息等等。 ls命令的常用参数有-a, -l等。

  • mkdir

    • 该命令用来创建指定的名称的目录，要求创建目录的用户在当前目录中具有写权限，并且指定的目录名不能是当前目录中已有的目录。

  • pwd

    • pwd可以用来查看”当前工作目录“的完整路径。简单地说，每当你在终端进行操作时，你都会有一个当前工作目录。

  • pgrep
  • pkill
  • tail

    • tail命令用途是依照要求将指定的文件的最后部分输出到标准设备，通常是终端，通俗讲来，就是把某个档案文件的最后几行显示到终端上， 假设该档案有更新，tail会自己主动刷新，确保你看到最新的档案内容。带参数时，我用的比较多的是tail -f filename, 这样可以监视filename文件增长。

  • telnet
  • tcpdump

    • tcpdump，即dump the traffic on a network，根据使用者的定义对网络上的数据包进行截获的包分析工具。tcpdump可以将网络中传送的数据包的“头”完全截获下来提供分析。 它支持针对网络层、协议、主机、网络或端口的过滤，并提供and、or、not等逻辑语句来帮助去掉无用的信息。

  • top

    • top命令是Linux下常用的性能分析工具，能够实时显示系统中各个进程的资源占用状况，类似于Windows的任务管理器。 top显示系统当前的进程和其他状况，是一个动态显示过程，即可以通过用户按键来不断刷新当前状态。 如果在前台执行该命令，它将独占前台，直到用户终止该程序为止。 比较准确的说，top命令提供了实时的对系统处理器的状态监视。它将显示系统中CPU最“敏感”的任务列表。该命令可以按CPU使用。内存使用和执行时间对任务进行排序； 而且该命令的很多特性都可以通过交互式命令或者在个人定制文件中进行设定。

  • ssh

    • ssh，即secure shell。用于远程登录linux主机，我用得最多的是ssh -i xxx.pem xxx@ip address。

  • source

    • 该命令的使用方法是：source filename。即在当前bash环境下读取并执行FileName中的命令。source命令通常可以用"."（点符号）来代替。 它以一个脚本为参数，该脚本将作为当前shell的环境执行，不会启动一个新的子进程。所有在脚本中设置的变量将成为当前Shell的一部分。

  • whereis
  • which
  • whoami
